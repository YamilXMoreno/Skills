# hilbench-pipeline

A single, self-contained skill package that runs the outcome_ladybug (HiL-Bench)
blocker-injection workflow as **segmented, reviewable, individually runnable stages**
instead of one ~50-minute autonomous run. It also provisions and verifies the SWEAP
container and runs the Attempter Checks as a real agent solve **inside** that container.

It is **self-contained**: it bundles the two grading skills (`input_validation` +
`evaluate_task`) it delegates to, so no separate `hilbench-evaluation` package is required.
It is also **additive** — it does not modify the older `hilbench-blocker-injection` /
`hilbench-evaluation` packages if they happen to be installed alongside it.

> New here? Start with the hands-on [`CONTRIBUTOR_GUIDE.md`](./CONTRIBUTOR_GUIDE.md) — a
> step-by-step walkthrough of running the pipeline. This README is the architecture/reference.
> Reviewing and fixing a task that already has a blocker set (a full `deliverables/` folder,
> whether generated by the pipeline or authored by an earlier contributor)? See
> [`REVIEWER_GUIDE.md`](./REVIEWER_GUIDE.md).

## Why this exists

The older `hilbench-blocker-injection` skill assumed the local Docker workflow had already
produced a repo at `/app` and did everything (steps 01-11) in one shot, with weak isolation
for the attempter checks (prompt discipline only). This package:

- Fills the repo-provisioning gap (docker pull + start + parent-commit verification).
- Splits the pipeline into ~15 commands, each of which STOPS for human review.
- Runs Check 1 / Check 2 as: scripted clean-state prep -> fresh subagent solve in the
  container -> scripted `task_checker.py` grading, so isolation comes from a scrubbed disk +
  scoped inputs + a clean-context subagent, not just prompt wording.

## Install

```bash
bash install.sh
```

Installs three skills into `~/.claude/skills/` (`hilbench-pipeline`, plus the bundled
`input_validation` and `evaluate_task`) and the `/hilbench-*` commands into
`~/.claude/commands/`. See `INSTALL.md` for custom roots and uninstall.

The evaluation commands (input validation + agentic checks) delegate to the bundled
`input_validation` / `evaluate_task` skills, so in-sandbox Harbor grading works from this
package alone — no separate `hilbench-evaluation` install needed.

## Environment model

- **Sandbox** (durable): `task_files/` inputs, `deliverables/` outputs, this skill, the
  scripts, and the model. Nothing important lives only in the container.
- **Container** (disposable SWEAP image): holds the `/app` repo at the parent commit plus
  the language toolchain. Provisioned up front; can be stopped during review and rebuilt
  from the image.
- **Bridge**: `docker cp` the checker + patches into the container and run `task_checker.py`
  there (it defaults cwd to `/app` and resets git each run).

## The stages (each STOPS for review)

| Command | Stage | Notes |
|---------|-------|-------|
| `/hilbench-provision` | 0. Provision | pull image once, start container, verify `base_commit_hash`, export `HILBENCH_*`. Hard-fails `PARENT_COMMIT_MISMATCH`. `--validate-original` opt-in chains stage 0.5. |
| `/hilbench-validate-original` | 0.5 Baseline | original test+golden patches apply + clean FAIL→PASS on the parent commit (Codebase Editing Workflow step 5). |
| `/hilbench-registry-generate` | 1a. Author (one-shot) | generator, from originals |
| `/hilbench-plan` + `/hilbench-registry` | 1a. Author (numbered) | step 01 -> 02 |
| `/hilbench-validate-registry` | 1b. **GATE** | schema/shape + verbatim model evals (distribution, self-ref, objective, independence, type, descriptions, questions, eval-7 B/C) + guessability/type-coherence linters; runs on every authoring path |
| `/hilbench-inject` | 2. Inject | steps 03-06 (modified artifacts + `relevant_tests.txt`, +07 obstructed patches on Scenario 2) |
| `/hilbench-validate-artifacts` | 3. **GATE** | step 08 patch-content validator + alignment map, plus verbatim patch-dependent evals: test_list (1), test_relevance (2), golden_patch (3), request (12), eval-7 Test A necessity |
| `/hilbench-validate-obstructed` | 3.5 Baseline | dynamic FAIL→PASS on the AUTHORED obstructed patches: bakes `setup_patch`, then runs the obstructed test+golden in the container. Grades your golden (not an agent patch) — the execution signal GATE 2's static check can't give. |
| `/hilbench-check1` | 4. Pre-screen (optional) | prep -> fresh subagent solve (no resolutions) -> eval; PASS = not guessable |
| `/hilbench-check2` | 4. Pre-screen (optional) | prep -> fresh subagent solve (with resolutions) -> eval; PASS = solvable |
| `/hilbench-validate-input` | 5. Input validation (optional; also run by `-full`) | delegates to `input_validation` |
| `/hilbench-evaluate-check1` / `-check2` / `-full` | 5. Agentic checks (**REQUIRED**, heavy) | delegates to `evaluate_task` (Harbor, 12 runs) |
| `/hilbench-teardown` | — | stop/remove container; keep image + deliverables |

Two layers of checks, on purpose:
- **Attempter Checks** (`/hilbench-check1/2`) — fast, local, single-model scripted F2P screens
  in the container. **Optional but recommended** as a cheap pre-screen before the required
  Harbor grade; they catch obvious guessable/unsolvable tasks without spending the heavy
  budget, but a single-model local PASS is not a substitute for the Harbor grade.
- **Harbor agentic checks** (`/hilbench-evaluate-*`) — the authoritative, **required** grade
  (multi-model, up to 12 SWE-Agent runs). Contributors run `/hilbench-evaluate-full`, then
  re-run an individual `/hilbench-evaluate-check1/2` to fix/recheck.

## Check 1 / Check 2 internals

`prepare_check.sh <mode>`:
1. `git reset --hard <base_commit>` in the container repo; clean untracked leftovers.
2. If `setup_patch.diff` exists, apply it and commit as the working baseline (so the
   checker's `git checkout -- .` resets to the setup-applied state).
3. Scrub the answer-key files (test/golden patches, relevant tests, registry) from the
   container.

Solve: a FRESH subagent (clean context) receives only the scoped inputs
(`references/attempter-check{1,2}.md`) and edits the real repo via `docker exec`.

`evaluate_check.sh <mode>`:
1. Capture the agent's changes as `<mode>_agent_patch.diff` (from `git diff` if not written).
2. `docker cp` the checker + obstructed test patch + relevant tests + agent patch into `/app`.
3. Run `task_checker.py --instance-id ... --golden-patch agent_patch.diff`.
4. Map the exit code:
   - Check 1: exit 0 -> `FAIL_GUESSABLE`; 10/11/12/1 -> `PASS`; 8/9 -> `CHECK_ERROR_PATCH`;
     others -> `CHECK_ERROR_ENV`.
   - Check 2: exit 0 -> `PASS`; 10/11/12/1 -> `FAIL_UNSOLVABLE`; 8/9 -> `CHECK_ERROR_PATCH`;
     others -> `CHECK_ERROR_ENV`.

A review pause between solve and eval lets the contributor inspect the agent patch.

## Repairing failures

See `references/optional-repair.md`: `FAIL_GUESSABLE` -> redesign the guessable blocker;
`FAIL_UNSOLVABLE` -> add the missing spec detail and regenerate the golden patch (never
weaken tests); `CHECK_ERROR_*` -> fix the diff/env and re-run only the eval step.
